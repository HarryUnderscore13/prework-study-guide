HTML

  • The head element contains information about the webpage, such as title and metadata
  • Anything that needs to be visible to the user goes in the body element, such as headings, images and text
  • All elements are enclosed in angled brackets
  • HTML is what defines the content of the webpage

CSS

  • CSS is what provides your webpage with style choices, such as font, colour, backgrounds and page layouts
  • A margin indicates how much space we want around an element
  • Padding indicates how much space we want around the content, within the elemetn
  • Style sheets must be linked to the HTML file with a LINK tag in the HEAD

Git

  • git status: checks what brance you are working in
  • git checkout -b branch-name: creates a new branch
  • git clone: retrieve an entire repo from a hosted location
  • git commit -m "descriptive message": commit your staged content as a new commit snapshop
  • git pull: fetch and merge any commits from the tracking remote branch

JavaScript

  • JavaScript is responsible for the interactivity of a webpage
  • JavaScript uses variables (a named container) that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
